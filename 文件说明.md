# 背单词应用 — 文件说明

## 文件一览

| 文件 | 作用 | 什么时候改它 |
|------|------|-------------|
| `index.html` | 页面骨架（标题、顶栏、卡片区域、进度条） | 增删页面元素 |
| `style.css` | 全部样式（颜色、字体、按钮、动画） | 改外观、换颜色、调间距 |
| `app.js` | 全部功能逻辑（卡片、评分、同步、登录） | 改学习流程、按钮行为 |
| `vocab_think_starter.js` | Think Starter 教材的词库数据 | 增删改这个教材的单词 |
| `vocab_waiyan_1.js` | 外研版必修一教材的词库数据 | 增删改这个教材的单词 |
| `vocab_detail.js` | 词的拓展知识点（词族、短语、例句、辨析） | 给某个单词补充拓展内容 |

---

## 每个文件的详细说明

### index.html — 页面骨架

- 只有 HTML 结构，不包含样式和逻辑
- 定义了三个区域：
  - **顶栏**（用户头像、标题、发音开关）
  - **进度条**（当前轮次、剩余词数）
  - **主卡片区**（单词卡片动态渲染在这）
- 通过 `<link>` 引入 `style.css`，通过 `<script>` 引入 `app.js` 和数据文件

**什么时候改它：** 几乎不用改。除非你想在页面上增加或删掉某个元素。

---

### style.css — 全部样式

- 颜色、字体、按钮形状、卡片动画、进度条、浮层等所有视觉效果
- 顶部 `:root` 块里定义了整套颜色变量（`--primary`、`--accent` 等），改一个颜色整页生效

**什么时候改它：**
- 想换配色 → 改 `:root` 里的颜色变量
- 按钮太小/太大 → 改对应的 `.btn-xxx` 样式
- 卡片动画太快/太慢 → 改 `transition` 或 `animation` 数值

---

### app.js — 全部功能逻辑

这是最大的文件，包含了应用的完整行为。大致分为以下几块：

| 功能区域 | 干什么的 |
|---------|---------|
| 登录/用户管理 | 多用户切换、云端同步、进度清除 |
| 选教材/选范围 | 选哪个教材、选哪些单元、认读/默写模式 |
| 学习主流程 | 英→中（认读）、中→英（拼写）两个阶段 |
| 评分 & 热力图 | 记录每个词的正确/错误，生成学习热力图 |
| 发音 | 英文单词朗读，可开/关 |
| 导出/导入 | 把学习进度导出为文件或从文件恢复 |
| 拓展知识点 | 点击"拓展"按钮查看词的词族、短语、例句 |
| 云端同步 | 把进度存到 Supabase，多设备同步 |

**什么时候改它：**
- 想改变某个按钮点了之后的行为
- 想增加新功能（比如错题复习、计时器）
- 修 bug

---

### vocab_think_starter.js — Think Starter 词库

- Cambridge Think Starter 教材的单词数据
- 每条词包含：英文、音标、中文、所属单元、是否认读词
- 格式示例：`{ en: "challenge", phonetic: "/ˈtʃælɪndʒ/", cn: "n.挑战", unit: "Unit 1", spell: true }`

---

### vocab_waiyan_1.js — 外研版必修一词库

- 外研版高中英语必修一的单词数据
- 格式和 Think Starter 完全一样

---

### vocab_detail.js — 拓展知识点

- 给部分单词补充了更丰富的信息：
  - **词族**（如 challenge → challenging、challenger）
  - **常用短语**（如 face a challenge、meet the challenge）
  - **例句**
  - **词义辨析**（近义词、反义词、使用场景区别）
- 点击学习卡片上的"拓展"按钮查看

---

## 日常操作速查

| 我想… | 改哪个文件 |
|--------|-----------|
| 换配色 | `style.css` 顶部 `:root` 块 |
| 改按钮大小 | `style.css` |
| 加新教材的词 | 新建 `vocab_xxx.js`，然后在 `index.html` 里加一行 `<script>` 引入 |
| 给某个词补充拓展 | `vocab_detail.js` |
| 修 bug / 改行为 | `app.js` |
